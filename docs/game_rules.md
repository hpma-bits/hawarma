# 游戏规则

> 本文档定义游戏的核心规则，作为模拟环境和验证的唯一依据。
> 不涉及自动化实现细节。

---

## 1. 游戏界面元素

```
┌────────────────────────────────────────────────────┐
│  订单区 (4 slot)                                    │
│  [slot0] [slot1] [slot2] [slot3]                   │
│                                                    │
│  灶台区 (4)          组装站 (1)                     │
│  [grill]             [assembly]                    │
│  [oven]                                          │
│  [skillet]                                         │
│  [pot]                                             │
│                                                    │
│  食材区 (8)         调料区 (8)                      │
│  [ing0]..[ing7]     [cond0]..[cond7]               │
│                                                    │
│  库存区 (3)         取餐台 (4)                      │
│  [stk0] [stk1] [stk2]  [pk0] [pk1] [pk2] [pk3]   │
│                                                    │
│  垃圾桶 (130, 560)                                  │
└────────────────────────────────────────────────────┘
```

## 2. 订单

### 2.1 订单属性

- **配方**：决定需要哪些食材、用哪个灶台、烹饪多久
- **调料偏好**：每种调料需要几份（1 或 2）
- **是否 rush**：rush 订单时限更短、得分更高
  - Rush 订单背景为**红色**，普通订单为其他颜色
  - 可通过检查订单卡特定位置像素颜色来识别
  - 红色在订单生命周期内保持一致

### 2.2 配方约束

- **食材唯一性**：每种食材在同一份菜品中最多出现 1 份，不存在需要 2 份相同食材的菜谱
- **食材数量**：一份菜品最多需要 2 种不同食材（各 1 份），部分菜谱仅需 1 种食材
- **食材-厨具绑定**：特定食材必须使用指定厨具烹饪，该组合决定唯一烹饪时长
- **配方唯一性**：两个不同的 recipe 不能包含相同的 ingredient-cooker 组合但不同的 duration

### 2.3 订单显示

- 最多 4 个订单同时显示（slot 0-3）
- 新订单出现在最左侧空 slot
- 如果 4 个 slot 都满了，新订单不会出现
- **订单生成间隔**：游戏开始后，每隔 4 秒自动刷新一个新订单
  - 第1个订单：游戏开始后第4秒出现
  - 第2个订单：游戏开始后第8秒出现
  - 以此类推，直到游戏结束或所有槽位已满
- **Rush 订单识别**：Rush 订单的订单卡背景为红色（普通订单为其他颜色），可通过检查订单卡特定位置的像素颜色来识别

### 2.4 Slot 位移

订单被提交后：
1. 该 slot 变为 None
2. 右侧所有订单左移填补空位
3. 位移需要约 1.5 秒动画时间
4. **状态立即更新**：内部状态在提交后立即更新（slot 0 立即包含原 slot 1 的订单），动画只是视觉效果
5. 动画期间限制：
   - 不能扫描订单区（会捕获未刷新的屏幕）
   - 不能送餐提交（assembly → pickup）
   - 可以进行其他操作：烹饪、调味、移到组装站、清理灶台等

示例：
```
提交前: [A, B, C, D]     slots: [0, 1, 2, 3]
提交 A: [_, B, C, D]  → 位移 → [B, C, D, _]    slots: [0, 1, 2]
提交 B: [_, C, D, _]  → 位移 → [C, D, _, _]    slots: [0, 1]
```

### 2.5 订单超时

- **时限**：普通订单约 70 秒，rush 订单约 40 秒
- 超时后订单失败，触发 slot 位移（同订单提交）
- **超时不影响 assembly station**：已放置的食材保留，可用于其他需要该食材的订单
- 只有符合 recipe 要求的食材组合才能成功提交，否则会被退回 assembly station

### 2.6 游戏时间限制

- **总游戏时长**：90 秒
  - 游戏从 0 秒开始，到 90 秒结束
  - 时间到 90 秒时游戏自动结束
  - 游戏结束后不再生成新订单
- **游戏目标**：在 90 秒内尽可能完成更多订单，获得更高分数

### 2.7 得分机制

- **基础得分**：完成订单获得基础分数
- **时间惩罚**：订单剩余时间越少，得分越低（顾客等待时间越长，满意度越低）
- **调料匹配**：添加的 condiment 与订单偏好匹配可获得更高分数
  - 必须添加 recipe 要求的 condiment 类型和数量
  - 无效 condiment（非 recipe 要求）不影响得分但也不加分
  - 调料总数量上限为 3 份，超过部分被忽略

## 3. 烹饪

### 3.1 流程

1. 将食材从食材区 swipe 到对应灶台
2. 等待烹饪完成（时间由配方决定）
3. 将成品从灶台 swipe 到组装站或库存区

### 3.2 灶台约束

- 每个灶台同时只能处理一个食材
- **并行烹饪支持**：多个灶台可以同时独立工作，各自维护自己的计时器
  - 例如：grill 烹饪牛肉（3秒），同时 oven 烹饪鱼（4秒）
  - 两个灶台完全独立，互不干扰
  - 可以通过 tick 推进时间，系统会自动检查每个灶台的完成状态
- 烹饪完成后，食材在灶台最多停留 **5 秒**
- 超过 5 秒未移走 → 食材过期（糊掉），必须 swipe 到垃圾桶 (130, 560)
- 灶台在食材被移走后才释放
- **可烹饪任意食材**：即使当前没有订单需要该食材，也可以烹饪。食材会一直留在灶台直到过期或被移走
- **烹饪时长**：根据 ingredient-cooker 组合从配方数据中查找，每个合法组合有唯一且固定的烹饪时长

## 4. 库存

### 4.1 机制

- 3 个库存 slot，每种食材最多存 **5 份**
- 会话开始时选出 3 种高频食材分配到库存 slot
- 空闲时可将烹饪好的食材存入库存（swipe 灶台 → 库存 slot）
- 需要时直接从库存 swipe 到组装站，跳过烹饪

### 4.2 库存约束

- **厨具绑定**：每个库存 slot 只能存放由**同一种厨具**烹饪的**同一种食材**
- **数量上限**：每个 slot 最多存 5 份
- **兼容性规则**：
  - 同种食材 + 不同厨具 = 不能共存于同一 slot
  - 不同食材 + 同种厨具 = 不能共存于同一 slot
- **失败处理**：尝试将不兼容的食材移入已有食材的 slot 时，操作失败，食材留在原处

## 5. 组装站

### 5.1 约束

- **同一时刻只能存放一份菜品**：组装站上最多存放一个完整 recipe 所需的食材组合
- **食材兼容性**：组装站上的食材必须属于同一个 recipe。如果尝试添加不同 recipe 的食材，该食材会被退回来源位置
- **自动组装**：当组装站集齐某个 recipe 所需的所有食材后，菜品自动完成组装，此时可以添加 condiment 或直接提交

### 5.2 流程

1. 食材逐一到达组装站
2. 所有食材到齐 → 进入调味阶段
3. 调料逐一从调料区 swipe 到组装站（双份 swipe 两次）
4. 全部完成后 swipe 组装站 → 取餐台

### 5.3 约束

- **食材上限**：组装站上每种食材最多 1 份，最多 2 种不同食材（单食材菜谱只需 1 种）
- **调料上限**：一份菜品最多 3 份调料
- **无效调料**：非 recipe 要求的调料 swipe 到组装站不会产生任何效果（菜品状态不变）
- **食材不绑定订单**：组装站上的食材不属于特定订单，可被任意需要该 recipe 的订单使用
- **失败退回**：尝试向组装站添加不兼容的食材时，该食材会被退回来源位置，不进入组装站
- **食材视觉**：玩家可以区分组装站上**不同种类**的食材（如区分牛肉和鸡蛋），但**无法区分同种食材用不同厨具烹饪**（如无法区分烤牛肉和煮牛肉）

## 6. 取餐台

- 4 个取餐台，与订单 slot 一一对应
- 组装站 → 取餐台[N] 的 swipe 表示将订单送到第 N 个取餐台
- 送餐后订单完成，slot 位移

## 7. 操作约束

| 规则 | 说明 |
|------|------|
| 每次只能执行一个 UI 操作 | swipe/click 必须串行 |
| 动画窗口限制 | slot 位移动画期间（~1.5s）：禁扫描订单区、禁送餐提交；其他操作不受限 |
| swipe 坐标必须合法 | 起点/终点必须是游戏元素位置 |
| 灶台过期清理 | 食材在灶台最多停留 5 秒 |
| 丢弃食材 | 可将 cooker、stockpile、assembly station 的食材丢弃到 trash |

## 8. 操作类型

| 操作 | 起点 | 终点 | 含义 |
|------|------|------|------|
| 烹饪 | 食材区[ing] | 灶台[cooker] | 开始烹饪 |
| 移到组装站 | 灶台[cooker] | 组装站 | 烹饪完成，送组装 |
| 移到库存 | 灶台[cooker] | 库存区[slot] | 烹饪完成，存库存 |
| 从库存取出 | 库存区[slot] | 组装站 | 跳过烹饪 |
| 调味 | 调料区[cond] | 组装站 | 添加调料 |
| 送餐 | 组装站 | 取餐台[slot] | 提交订单 |
| 丢弃 | 灶台/库存/组装站[食材] | 垃圾桶 | 主动丢弃任意位置的食材 |
| 清理 | 灶台[cooker] | 垃圾桶 | 清理过期食材（等同丢弃） |
