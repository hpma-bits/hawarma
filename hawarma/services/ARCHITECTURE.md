# hawarma/services 目录架构

## 📁 目录概述

此目录包含烹饪Bot的服务层组件。架构重构后，服务层分为：
- **检测服务**: 订单检测
- **执行服务**: 原子UI动作执行
- **资源守卫**: 物理资源锁（不含业务策略）

## ⚠️ 重要提示

**一旦本目录有变化（新增/删除/重命名文件），请立即更新本文档！**

## 📄 文件列表

### `__init__.py`
- **地位**: 包初始化文件
- **导出**: `DetectionService`, `Executor`, `ResourceGuards`

### `detection_service.py`
- **地位**: 订单检测服务
- **状态**: ✅ 保持不变
- **功能**:
  - 从屏幕截图检测客户订单
  - 识别配方（食材+厨具组合识别）
  - 检测加急订单状态
  - 检测调料偏好
- **输入**: 配置对象、配方列表、屏幕截图
- **输出**: Order对象或None

### `executor.py`
- **地位**: 原子动作执行器（新增）
- **状态**: ✅ 完成
- **功能**:
  - 执行Scheduler返回的Action对象（Scheduler已完成所有决策和资源预留）
  - 执行物理UI操作（swipe）
  - UI成功后提交结果性状态（入assembly、扣库存、完成订单）
  - 负责reservation生命周期释放（release_prep/release_finish）
- **设计原则**:
  - Executor不做重复检查，所有决策由Scheduler完成
  - 只负责物理UI操作和结果状态提交
- **输入**: Action对象（已由Scheduler预留资源）
- **输出**: 动作执行结果、状态更新
- **关键方法**:
  - `execute()` - 执行单个动作
  - `execute_batch()` - 执行动作批次
  - `_cook_ingredient()` - 烹饪并移动食材（Scheduler已预留资源）
  - `_pull_from_stockpile()` - 从库存拉取（Scheduler已预留资源）
  - `_finish_order()` - 调味并上菜（Scheduler已预留资源）

### `resource_guards.py`
- **地位**: 物理资源锁（新增）
- **状态**: ✅ 完成
- **功能**:
  - 管理烹饪设备锁
  - 管理库存槽位锁
  - 管理组装台锁
- **输入**: 资源请求
- **输出**: 锁的获取/释放
- **注意**: 不包含任何业务策略

### `recipe_manager.py`
- **地位**: 配方管理服务
- **状态**: ✅ 保持不变
- **功能**: 加载和管理配方数据

## 🔗 模块间关系

```
┌─────────────────────────────────────────┐
│         CookingBotApp (app.py)          │
│            生命周期管理器                  │
└────────────────┬────────────────────────┘
                 ↓
┌─────────────────────────────────────────┐
│              Scheduler                   │
│  - OrderPolicy (订单优先级)               │
│  - StockpilePolicy (库存策略)             │
└────────────────┬────────────────────────┘
                 ↓
┌─────────────────────────────────────────┐
│              Executor                    │
│  执行Action对象，更新GameState            │
│  自动处理阶段转换                         │
└────────────────┬────────────────────────┘
                 ↓
┌──────────────┐         ┌───────────────┐
│ResourceGuards│         │UIOperationMgr │
│ 物理资源锁   │         │  全局UI锁      │
└──────────────┘         └───────────────┘
```
